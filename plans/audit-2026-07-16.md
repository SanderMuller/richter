## Finding details

### [1] Diff parser mistakes hunk-body lines starting `++ `/`-- ` for file headers

- **Evidence**: `src/Changes/UnifiedDiffParser.php:44-62` (`--- `/`+++ ` header branches) run for every line with no "inside a hunk body" state; content branches at `:74-80`. An *added* content line whose text starts with `++ ` arrives as diff line `+++ …` and matches the header branch at `:50`, reassigning `$current` to a bogus path; a *removed* line starting `-- ` arrives as `--- …`, matches `:44`, and is silently dropped while polluting `$pendingOld`.
- **Impact**: In `ChangedSymbols::classifyFile` (`src/Changes/ChangedSymbols.php`) the real file then has empty/incomplete hunks → can classify `cosmeticOnly` → seeds nothing → the change reads as **"no impact"** — exactly the falsely-empty outcome the tool exists to prevent. Realistic triggers: SQL `-- comments` in heredocs/nowdocs, markdown in string literals, spaced `++ $x;` statements.
- **Effort**: S · **Risk**: LOW · **Confidence**: HIGH (traced end-to-end by lead reviewer)
- **Fix sketch**: Track hunk state — set an `inHunk` flag on `@@`, clear on `diff --git`; while in a hunk, route `+`/`-` lines only to the content branches.

### [2] MCP `detect-changes` tool doesn't catch `InvalidArgumentException`

- **Evidence**: `src/Mcp/Tools/DetectChangesTool.php:40` calls `RichterConfig::baseRef($request->get('base'))` *outside* the `try` at `:42-46`, which catches only `RuntimeException`. `RichterConfig::refOrFail` (`src/Support/RichterConfig.php:82-89`) throws `InvalidArgumentException` on a leading-`-` ref. The Artisan command handles exactly this (`src/Console/DetectChangesCommand.php:77`) with a regression test (`tests/Feature/CommandsTest.php:171-184`); the MCP tool has neither.
- **Impact**: An MCP client passing an option-shaped base (e.g. `--upload-pack=…`) gets an uncaught exception instead of a clean `Response::error`. The injection itself is blocked by `refOrFail` — this is an error-surface bug, not an injection hole.
- **Effort**: S · **Risk**: LOW · **Confidence**: HIGH
- **Fix sketch**: Move `baseRef` inside the try and widen the catch to `InvalidArgumentException|RuntimeException`, mirroring the command. Add MCP success-path tests while there (see finding 5's sibling gap: `tests/Feature/McpTest.php` currently asserts only registration + two error strings).

### [3] README "Nothing runs" guarantee is false — analysis autoloads analyzed app code

- **Evidence**: `README.md:29` — "Nothing runs: it is static analysis over a code graph, so it is … **safe on code you would not want to execute**." Meanwhile the default build path uses reflection that triggers the host autoloader: `src/Support/AppFiles.php:92,96` (`class_exists` + `constant()` — the constant initializer expression is evaluated), `src/Tracers/DispatchEdgeTracer.php:255` (`class_exists && is_subclass_of` over every `app/` file), `src/Tracers/EagerLoadStringChecker.php:244,260` (`class_exists` + `get_class_methods` on every model), `src/Graph/CodeGraphBuilder.php:306` (`method_exists` on controllers).
- **Impact**: Running `richter:detect-changes` (or the MCP tools) on a checkout autoloads that checkout's class files: top-level statements and constant initializers execute with the CI runner's / analyst's privileges. The documented use case — triage PR branches in CI — is precisely where the promise gets relied on. Documented-guarantee-vs-implementation drift: doc or code must change.
- **Effort**: S for the doc correction; L for full AST-only resolution · **Risk**: MED on the static path (accuracy loss in relation-name/queueable checks) · **Confidence**: HIGH
- **Fix sketch**: Decide the contract. Cheapest honest fix: correct `README.md:29` to state class definitions are autoloaded and the tool is not safe against hostile code. Full fix: replace reflection with resolution from the already-parsed ASTs, or gate reflection behind opt-in with static-only default.

### [4] Pure renames are invisible to the diff parser

- **Evidence**: `src/Changes/UnifiedDiffParser.php:36-62` reacts only to `diff --git`, `--- `, `+++ `, `@@`, `+`, `-`. A 100%-similarity rename emits `similarity index 100%` + `rename from`/`rename to` and **no** `---`/`+++`/`@@` lines (rename detection is on by default in git; `ChangedSymbols::resolve` runs plain `git diff -U0 --end-of-options` at `src/Changes/ChangedSymbols.php:23`), so no file entry is parsed and `ChangedSymbols::resolve` iterates nothing.
- **Impact**: Moving a class file without editing it (old FQCN disappears — every caller breaks) reports as no changed files / no impact. Scoped to content-identical renames; moves that edit the `namespace` line still emit hunks and are handled.
- **Effort**: M · **Risk**: LOW · **Confidence**: HIGH on mechanism, MED on real-world frequency
- **Fix sketch**: Parse `rename from`/`rename to` (and `copy from`/`copy to`); when a section has no hunks, register the file with a coarse class-level change (old path as `oldPath`) so the FQCN change is not read as cosmetic.

### [5] Interface-implementation and container-binding edges have zero test coverage

- **Evidence**: `src/Tracers/EntryPointTracer.php:236-285` (`bindingEdges()`, `interfaceEdgesForResolvedAst()`) is non-trivial pure logic. The fixture app (`tests/Fixtures/project`) contains no `interface`, no `->bind(`, no `->singleton(` (verified by grep), so `tests/Feature/CodeGraphBuilderTest.php` never touches either path. `README.md:18` headlines both edge types as Richter's core advantage over Brain.
- **Impact**: Two marquee correctness claims can silently break with zero CI signal.
- **Effort**: M · **Risk**: LOW · **Confidence**: HIGH
- **Fix sketch**: Add a contract + implementor + provider binding to the fixture project; assert the `implements`/`binding` edges appear and that a caller/dependency walk crosses them.

### [6] Code graph rebuilt from scratch on every command/MCP call

- **Evidence**: `src/Mcp/Tools/ImpactTool.php:43` and `src/Mcp/Tools/DetectChangesTool.php:52` call `$this->builder->build()` per tool call; same per command run (`src/Console/ImpactCommand.php:29,43`, `src/Console/DetectChangesCommand.php:94,152`). `CodeGraphBuilder::build()` (`src/Graph/CodeGraphBuilder.php:41`) has no memoization; no cache exists anywhere in `src/`. `src/Console/BenchmarkCommand.php:48` builds once and reuses the graph across all cases — proving reuse is safe.
- **Impact**: The full build (Brain's whole-app `ProjectAnalyzer::analyze`, a consolidated per-file AST pass, entry-point tracing, two Blade walks) is the dominant wall-clock cost. In the long-lived MCP server, an agent running three `impact` queries pays three full rebuilds — tens of seconds each on a large host app.
- **Effort**: M · **Risk**: MED — cross-process caching must invalidate on tree changes or it reports stale impact (the exact false reassurance the tool exists to avoid); in-process memoization is trivial · **Confidence**: HIGH (rebuild-per-call), MED (absolute cost, unprofiled)
- **Fix sketch**: Memoize the `CodeGraph` on the builder instance keyed by project root (covers the MCP process). On-disk persistence keyed by HEAD sha + dirty-tree guard is a separate design spike (see Direction).

### [7] `laravel/mcp` 0.x soft dependency unconstrained for consumers

- **Evidence**: `composer.json:35` has `laravel/mcp: ^0.8` in require-dev only; `composer.json:121-123` suggests it with no version bound. Shipped dist references its API in four files (`src/RichterServiceProvider.php:5`, `src/Mcp/RichterServer.php:5-6`, `src/Mcp/Tools/*.php`). The only guard is `class_exists(Mcp::class)` (`src/RichterServiceProvider.php:29`) — it protects the *not-installed* case only.
- **Impact**: A consumer installing `laravel/mcp` themselves (the README-documented path) can get a future 0.9/1.0 whose renamed/reshaped API passes the guard and fatals at boot. 0.x releases routinely break between minors.
- **Effort**: S · **Risk**: LOW · **Confidence**: HIGH
- **Fix sketch**: Publish a visible version boundary (a `conflict` block excluding incompatible ranges, or a documented supported range) plus a boot smoke test in the MCP-installed CI cell.

### [8] `build()` mutates Brain config globally without restore

- **Evidence**: `src/Graph/CodeGraphBuilder.php:45-48` — four `config()->set('laravel-brain.…')` calls before analysis, no save/restore.
- **Impact**: In the long-lived MCP process the host app's Brain config stays permanently overridden after the first build; anything else reading Brain config in-process sees Richter's widened globs.
- **Effort**: S · **Risk**: LOW · **Confidence**: HIGH
- **Fix sketch**: Snapshot the four keys, set, restore in `finally`.

### [9] `resolveName` and NameResolver construction duplicated

- **Evidence**: Canonical helper `src/Support/AppFiles.php:81-86`; re-implementations at `src/Tracers/DispatchEdgeTracer.php:236`, `src/Tracers/PolicyEdgeTracer.php:105`, `src/Tracers/EntryPointTracer.php:223,289`. Identical `new NameResolver(null, ['preserveOriginalNames' => true, 'replaceNodes' => false])` at `AppFiles.php:75`, `EntryPointTracer.php:173`, `EagerLoadStringChecker.php:78`.
- **Impact**: A future name-resolution change silently drifts wherever a copy is missed → subtly wrong edges in one tracer only.
- **Effort**: S · **Risk**: LOW · **Confidence**: HIGH
- **Fix sketch**: Route every site through `AppFiles::resolveName()` / a single shared parse helper; delete the private copies.

### [10] Benchmark accuracy net ships empty and never runs in CI

- **Evidence**: `config/richter.php:29-37` defaults `benchmark_cases` to `[]`; `.github/workflows/run-tests.yml` runs PHPUnit only; `tests/Feature/CommandsTest.php` covers only no-cases/no-match/skip/fail branches. `tests/Unit/BenchmarkCaseTest.php` covers scoring, but no committed fixture or CI step exercises an end-to-end replay. `README.md:102-108` positions the benchmark as the trustworthiness regression net.
- **Impact**: The mechanism built to catch accuracy regressions in graph/tracers never executes.
- **Effort**: M · **Risk**: MED — SHA-pinned cases need `fetch-depth: 0` and can drift · **Confidence**: HIGH
- **Fix sketch**: Commit at least one signal case and one control resolvable in CI history; wire `richter:benchmark` into a CI job or a Feature test.

### [11] Diff-parser edge cases untested: CRLF, binary, mode-only

- **Evidence**: `src/Changes/UnifiedDiffParser.php:36` splits strictly on `"\n"` (CRLF leaves trailing `\r` in captured text); `tests/Unit/UnifiedDiffParserTest.php` covers modification/rename-with-edit/multi-line/multi-file/no-newline/deletion but not CRLF, binary hunks, or mode-only changes.
- **Impact**: CRLF currently survives by luck (`ChangedSymbols::normalize()` strips `\s`), unpinned by any test; binary/mode-only hunks are dropped without a test proving intent. A future header-regex tightening could regress silently on Windows-authored diffs.
- **Effort**: S · **Risk**: LOW · **Confidence**: MED (characterization-test candidate)
- **Fix sketch**: Add parser cases for a CRLF diff, a binary-file diff, and a mode-only change. Natural companion to the fixes for findings 1 and 4 (same file).

### [12] `nodesContaining` linear-scans every node per seed lookup

- **Evidence**: `src/Graph/CodeGraph.php:47-59` builds a boundary regex and filters all node keys per call; `src/Analysis/ImpactAnalyzer.php:307-335` calls it once per changed member/seed — O(members × nodes) on wide diffs against large graphs.
- **Impact**: Wide refactor diffs on big host apps pay a quadratic-ish seed-resolution cost on top of the build.
- **Effort**: M · **Risk**: LOW — boundary-match semantics (`Video` ≠ `VideoContainer`) must be preserved exactly · **Confidence**: MED (size-dependent)
- **Fix sketch**: Precompute a token → node-id index at graph construction; resolve seeds against it.

### [13] Brain coupling on undocumented internals, no contract test

- **Evidence**: `src/Graph/CodeGraphBuilder.php:45-48` hard-codes Brain's private config keys; `:50-72` reads `ProjectAnalyzer` graph internals (`fullGraph->nodes()/edges()`, `$node->id/data`, `Edge` fields). Further internal imports in `src/Graph/BladeViews.php:5` and `src/Tracers/EntryPointTracer.php:5-7`. Constraint `^2.3.1` allows any 2.x; no test asserts the node/edge/config contract.
- **Impact**: A Brain minor that renames a config key or reshapes node data silently produces an empty or wrong graph — the failure mode the tool's own comments acknowledge (`CodeGraphBuilder.php:55`).
- **Effort**: M · **Risk**: LOW · **Confidence**: HIGH
- **Fix sketch**: Narrow the constraint to the validated minor range; add a characterization test over `ProjectAnalyzer` output shape; consider one adapter seam for the three Brain-touching files.

### [14] Entry-point classes parsed ≥3× per build

- **Evidence**: `src/Graph/CodeGraphBuilder.php:78` runs `EntryPointTracer::trace()` separately from the consolidated pass (`:131-132`); `src/Tracers/EntryPointTracer.php:70,127-145` re-parses each entry-point class (`methodsOf`), `:156-166` parses `EventServiceProvider` again; Brain's `MethodTracer` re-parses downstream callees.
- **Impact**: Every class under the entry-point roots is parsed by Brain, by the consolidated pass, and by `methodsOf`; the docblock at `CodeGraphBuilder.php:83` records this class of cost (~30-60s per redundant pass on large apps).
- **Effort**: L — the deep re-parse lives inside Brain's `MethodTracer` · **Risk**: MED · **Confidence**: MED (Richter-side redundancy certain; magnitude unprofiled)
- **Fix sketch**: Feed `methodsOf()` and the `$listen` parse from the consolidated AST cache; investigate whether `MethodTracer` can accept pre-parsed ASTs.

### [15] Abandoned dev dependency `symplify/phpstan-extensions`

### [16] QA workflow undocumented; `composer qa` rewrites the working tree

### [17] Per-file AST traversed 5-6× by separate NodeFinder walks

- **Evidence**: `src/Graph/CodeGraphBuilder.php:145-151` invokes four tracers per AST; each does its own full-tree walks (`ReferenceEdgeTracer.php:75,93,117-119`; `DispatchEdgeTracer.php:79`; `PolicyEdgeTracer.php:52`; `EntryPointTracer.php:267`) — parsing is shared since the consolidation, traversal is not.
- **Impact**: Constant-factor traversal cost multiplied over every app file; smaller absolute win than findings 6/14.
- **Effort**: M · **Risk**: MED — touches correctness-sensitive tracers · **Confidence**: MED

- **Impact**: A relation added mid-MCP-session can produce a false "not a method on any model" finding on a later report. Narrow window; the memo is a documented cost tradeoff.
- **Effort**: S · **Risk**: LOW · **Confidence**: LOW — investigate before fixing

Honest counterpoint: LLM consumers read prose fine, and structured content is a second contract to version.

### [D4] `richter:benchmark:add <fix-commit>` capture scaffolder — effort M, grounding MED

Pairs with finding 10. The benchmark stays at `[]` because capturing a case is manual archaeology (`config/richter.php:29-37`, README `:102-108`). A scaffolder that resolves a fix commit and pre-fills the config stanza turns "I should record this regression" into one command.

## Considered and rejected

- **Refactoring the 4 baselined complexity hotspots** (`ImpactAnalyzer::detectChanges` 27, `TestReferenceIndex::addSource` 26, `ChangedSymbols::classifyFile` 23, `ReferenceEdgeTracer::relationsLoadedIn` 21): all have dedicated unit tests; none blocks testing or correctness. Cosmetic on a 4.5K-LOC actively-evolving repo. If ever touched, extract `classifyFile`'s branches only, opportunistically, with per-branch tests.
- **Console vs MCP pipeline duplication**: not present — both surfaces share `CodeGraphBuilder` → `ImpactAnalyzer` → presenters; the tools are thin wrappers. Good separation.
- **Git command/argument injection**: checked and clean — array argv (no shell), `--end-of-options` everywhere, leading-`-` refs rejected (`RichterConfig::refOrFail`). `BenchmarkCase.fixCommit` bypasses `refOrFail` but is config-trusted and still `--end-of-options`-guarded.
- **MCP tool input injection**: clean — symbol is `preg_quote`'d, base ref goes through `baseRef`.
- **ReDoS**: regexes reviewed (diff header, Blade, policy/middleware/controller-id, test-reference scanning) — negated character classes / non-overlapping delimiters, no catastrophic backtracking.
- **Graph BFS hazards**: cycles/self-references/empty graphs/depth handled via first-visit `seen` set (`CodeGraph.php:137-168`).
- **`catch (Throwable) { return null }` sites**: the documented honest-degradation design, not swallowed errors.
- **Docs staleness**: README config table, publish tag, version requirements, CI recipe all verified accurate. Only doc issue found is finding 3 (the "Nothing runs" claim) and finding 16 (QA workflow absent).
- **Core dependency currency**: Laravel 12/13, PHP 8.4, php-parser 5, symfony/finder 7/8 all current — no migration debt.
- **Double-parse of head source in `classifyFile` / Blade tree walked twice**: real but low-leverage (bounded by diff size / view-tree size); fold into finding 17 only if that refactor touches the same code.

**New finding from plan-011 execution (2026-07-17) — [19] Container-binding edges are silently absent for strict-typed providers:**
- **Evidence**: `vendor/laramint/laravel-brain/src/Analysis/ContainerBindingAnalyzer.php:69-72` unwraps the file's namespace only when `$stmts[0] instanceof Namespace_`; a leading `declare(strict_types=1);` makes the AST `[Declare_, Namespace_]`, so the provider is silently skipped and yields zero binding records. Proven during plan 011: the plan's `AppServiceProvider` fixture (with declare) produced no binding edge; an identical probe without the declare produced exactly the expected record. - **Impact**: The README's "container bindings" marquee edge is non-functional for strict-typed codebases — bindings resolve to nothing, silently, which is the falsely-empty result the package exists to prevent. Richter's own `bindingEdges()` (`src/Tracers/EntryPointTracer.php:236-251`) faithfully forwards the empty analyzer result.
- **Effort**: S (Richter-side workaround: pre-strip/skip the leading `Declare_` before handing sources to the analyzer is not possible — the analyzer reads files itself; realistic options are an upstream laravel-brain fix (skip `Declare_` nodes before the namespace unwrap) or a Richter-side replacement of `bindingEdges()` with its own consolidated-pass scan) · **Risk**: LOW · **Confidence**: HIGH (reproduced with probe + AST dump)
- **Status**: Richter-side workaround shipped via plan 012 (`advisor/012-native-binding-edges`, commit `6c3ed4c`): `bindingEdges()` now scans providers natively with `AppFiles::parseResolved()`, Brain-parity semantics, regression-tested against a strict-typed fixture provider. The upstream laravel-brain fix was PR'd on 2026-07-18: [laramint/laravel-brain#65](https://github.com/laramint/laravel-brain/pull/65) (iterate to the first `Namespace_` in `scanFile()`, regression-tested against a dedicated strict-types fixture project). Relates to finding 13 (one Brain coupling point removed).
