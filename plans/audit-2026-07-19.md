## Direction findings → plans

- **DIR-1** Benchmark corpus + CI replay — the evidence engine every parked decision
  (security phase 2, risk-floor namespaces, entry-point widening) is explicitly
  waiting on; tooling exists, corpus and process don't. → plan 029 (spike).
- **DIR-2** Livewire/Filament test-selection idioms. Vetting narrowed the premise
  twice: the generic class-reference scan (imports + qualified in-body refs,
  `TestReferenceIndex.php:100-112`) already covers the class-shaped idioms,
  **and** existing tests already pin them
  (`tests/Unit/TestReferenceIndexTest.php:213-228` — found post-tabling; the
  auditor's "no Livewire coverage" claim was src-only true, test-false). The
  one real gap: string-named components (`Livewire::test('admin.dashboard')`)
  map to nothing. → plan 030 (scoped to that gap).
- **DIR-3** Configurable risk-floor namespaces — grounded asymmetry
  (`entry_point_roots` configurable for tracing, risk floor fixed) but moves gate
  inputs; **deliberately not planned** until plan 029's corpus exists. Recorded
  here so it isn't re-derived.
- **DIR-4** `anyOf()` MCP schema — blocked on dropping the Laravel 12 floor
  (in-code comment in `DetectChangesTool`); track with the next major, not
  planned.
- **DIR-5** README PR-comment recipe (`--markdown` + `gh pr comment`) — GO'd
  docs-only in the handoff research; small, unplanned this round; fold into the
  next docs pass.

## Considered and rejected this round

- **ImpactAnalyzer god-file extraction** (626 lines, every annotation lane): the
  lanes are cohesive and the complexity hotspot is a decided/baselined tradeoff;
  the only seam worth extracting (annotation assembly) becomes worthwhile at the
  *next* annotation lane, not before. Reported by the debt auditor and downgraded
  by their own analysis; concur.
- **Unknown static route names silently dropping** (frontend): documented design
  ("unmatched names never guess") — only the *dynamic*-argument escape (F4) is a
  bug.
- **`generated_paths` traversal bypass**: unreachable — file paths come from git
  diff, repo-relative and normalized.
- **`constant()`/`class_exists` autoload of diff-derived names in
  FeatureGateChecker**: established, documented convention across the codebase
  (AppFiles, EagerLoadStringChecker, DispatchEdgeTracer, NodeMetadata); the tool's
  autoload caveat is in the README since finding 3's fix.
- **`copy from`/`copy to` diff sections**: need non-default git config to appear;
  noted in plan 021's out-of-scope.
